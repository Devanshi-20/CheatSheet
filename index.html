Day 3

INSERT, UPDATE, DELETE Operations

-- INSERT - Add new data INSERT INTO employees (id, name, salary) VALUES (1, 'John Doe', 75000); -- UPDATE - Modify existing data UPDATE employees SET salary = 80000 WHERE id = 1; -- DELETE - Remove data DELETE FROM employees WHERE id = 1;

⚠️ Critical Safety Tips

  • Always use WHERE clause with UPDATE/DELETE to avoid changing all rows
  • Test with SELECT first: SELECT * FROM table WHERE condition
  • Use transactions for critical operations

REGEXP (Regular Expressions)

-- Pattern Matching with REGEXP SELECT * FROM customers WHERE email REGEXP '^[a-z]'; -- Starts with lowercase letter SELECT * FROM products WHERE sku REGEXP '[0-9]{5}'; -- Contains 5 digits SELECT * FROM employees WHERE phone REGEXP '^\\+1'; -- US phone numbers

📝 Common REGEXP Patterns

^ Start of string $ End of string
. Any single character * 0 or more occurrences
+ 1 or more occurrences [abc] Any character in brackets

Date & Time Functions

-- Current Date/Time SELECT NOW() AS current_datetime, CURDATE() AS current_date, CURTIME() AS current_time; -- Date Arithmetic SELECT DATE_ADD(NOW(), INTERVAL 7 DAY) AS next_week, DATE_SUB(NOW(), INTERVAL 1 MONTH) AS last_month, DATEDIFF('2024-12-31', NOW()) AS days_until_newyear; -- Extract Date Parts SELECT YEAR(order_date) AS order_year, MONTH(order_date) AS order_month, DAY(order_date) AS order_day, DAYNAME(order_date) AS day_of_week FROM orders; -- Format Dates SELECT DATE_FORMAT(NOW(), '%Y-%m-%d %H:%i:%s') AS formatted_date;

📅 Common Date Patterns

  • %Y - Year (4 digits), %y - Year (2 digits)
  • %m - Month (01-12), %M - Month name
  • %d - Day (01-31), %W - Weekday name
  • %H - Hour (00-23), %i - Minute, %s - Second

Day 11

Subqueries

-- Subquery in WHERE clause SELECT name, salary FROM employees WHERE salary > ( SELECT AVG(salary) FROM employees ); -- Subquery in SELECT (Scalar Subquery) SELECT name, salary, (SELECT AVG(salary) FROM employees) AS company_avg FROM employees; -- Subquery with IN SELECT * FROM products WHERE category_id IN ( SELECT id FROM categories WHERE name = 'Electronics' ); -- Correlated Subquery SELECT e1.name, e1.salary FROM employees e1 WHERE salary > ( SELECT AVG(e2.salary) FROM employees e2 WHERE e2.department = e1.department );

💡 Subquery Types

  • Scalar: Returns single value
  • Row: Returns single row
  • Table: Returns multiple rows/columns
  • Correlated: References outer query

PIVOT, UNPIVOT & Data Transformation

-- Manual PIVOT using CASE (MySQL doesn't have native PIVOT) SELECT employee_id, SUM(CASE WHEN month = 'January' THEN sales ELSE 0 END) AS Jan, SUM(CASE WHEN month = 'February' THEN sales ELSE 0 END) AS Feb, SUM(CASE WHEN month = 'March' THEN sales ELSE 0 END) AS Mar FROM monthly_sales GROUP BY employee_id; -- UNPIVOT using UNION SELECT product_id, 'Q1' AS quarter, q1_sales AS sales FROM quarterly_sales UNION ALL SELECT product_id, 'Q2', q2_sales FROM quarterly_sales UNION ALL SELECT product_id, 'Q3', q3_sales FROM quarterly_sales UNION ALL SELECT product_id, 'Q4', q4_sales FROM quarterly_sales;

Day 13

Common Table Expressions (CTE)

-- Simple CTE WITH high_earners AS ( SELECT * FROM employees WHERE salary > 80000 ) SELECT * FROM high_earners WHERE department = 'Engineering'; -- Multiple CTEs WITH sales_summary AS ( SELECT employee_id, SUM(amount) AS total_sales FROM sales GROUP BY employee_id ), top_performers AS ( SELECT * FROM sales_summary WHERE total_sales > 100000 ) SELECT e.name, tp.total_sales FROM top_performers tp JOIN employees e ON tp.employee_id = e.id; -- Recursive CTE (Hierarchical Data) WITH RECURSIVE employee_hierarchy AS ( SELECT id, name, manager_id, 1 AS level FROM employees WHERE manager_id IS NULL UNION ALL SELECT e.id, e.name, e.manager_id, eh.level + 1 FROM employees e JOIN employee_hierarchy eh ON e.manager_id = eh.id ) SELECT * FROM employee_hierarchy;

🎯 CTE Benefits

  • ✅ More readable than nested subqueries
  • ✅ Can be referenced multiple times in same query
  • ✅ Supports recursive operations (org charts, trees)
  • ✅ Better for complex multi-step transformations
Day 14

Window Functions

📊
Ranking Functions
ROW_NUMBER, RANK, DENSE_RANK, NTILE
📈
Aggregate Windows
SUM, AVG, COUNT with OVER clause
⬅️➡️
Offset Functions
LAG, LEAD, FIRST_VALUE, LAST_VALUE
-- ROW_NUMBER - Assign unique sequential numbers SELECT name, salary, ROW_NUMBER() OVER (ORDER BY salary DESC) AS salary_rank FROM employees; -- RANK & DENSE_RANK - Handle ties differently SELECT name, score, RANK() OVER (ORDER BY score DESC) AS rank, DENSE_RANK() OVER (ORDER BY score DESC) AS dense_rank FROM students; -- PARTITION BY - Rank within groups SELECT department, name, salary, RANK() OVER ( PARTITION BY department ORDER BY salary DESC ) AS dept_rank FROM employees; -- Running Total SELECT order_date, amount, SUM(amount) OVER ( ORDER BY order_date ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW ) AS running_total FROM orders; -- LAG & LEAD - Access previous/next rows SELECT product_name, sales, LAG(sales, 1) OVER (ORDER BY sale_date) AS previous_sales, LEAD(sales, 1) OVER (ORDER BY sale_date) AS next_sales FROM product_sales;

Window Functions Concept

Perform calculations across rows related to current row without grouping

Example: Running Total
Date       | Amount | Running Total
-----------|--------|---------------
2024-01-01 |   100  |   100  ← SUM of row 1
2024-01-02 |   150  |   250  ← SUM of rows 1-2
2024-01-03 |   200  |   450  ← SUM of rows 1-3

-- Complex Real-World Query Example WITH monthly_revenue AS ( SELECT DATE_FORMAT(order_date, '%Y-%m') AS month, SUM(total_amount) AS revenue FROM orders WHERE status = 'completed' GROUP BY DATE_FORMAT(order_date, '%Y-%m') ) SELECT month, revenue, LAG(revenue) OVER (ORDER BY month) AS prev_month, ROUND( (revenue - LAG(revenue) OVER (ORDER BY month)) / LAG(revenue) OVER (ORDER BY month) * 100, 2 ) AS growth_percentage FROM monthly_revenue ORDER BY month;
